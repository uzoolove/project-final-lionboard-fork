# 요구사항 정의서

## 1. 문서 정보

- 문서 목적: 게시판 앱의 요구사항을 명확히 정의하여 이해관계자 간 공통 이해를 확보한다.
- 대상: 기획, 디자인, 프론트엔드, 백엔드
- 범위: 웹 기반 커뮤니티 게시판

## 2. 시스템 개요

사용자가 정보/자유/Q&A 게시판에서 글을 작성·조회·수정·삭제하고, 댓글과 검색 기능을 통해 소통하는 커뮤니티 앱이다. 로그인 사용자만 작성 기능을 이용할 수 있으며 비회원은 조회만 가능하다.

## 3. 사용자 유형

- 비회원: 게시판 목록 및 게시글 조회만 가능
- 회원: 게시글/댓글 작성 및 관리 가능

## 4. 기능 요구사항

| ID | 우선순위 | 요구사항 |
| --- | --- | --- |
| FR-01 | 1순위 | 사용자는 이메일과 비밀번호로 회원가입할 수 있어야 한다. |
| FR-02 | 1순위 | 사용자는 이메일과 비밀번호로 로그인할 수 있어야 한다. |
| FR-03 | 1순위 | 사용자는 정보/자유/Q&A 게시판 중 하나를 선택해 접근할 수 있어야 한다. |
| FR-04 | 1순위 | 사용자는 게시판별 글 목록을 조회할 수 있어야 한다. |
| FR-05 | 1순위 | 사용자는 게시글의 상세 내용을 조회할 수 있어야 한다. |
| FR-06 | 1순위 | 로그인 사용자는 게시글을 작성할 수 있어야 한다. |
| FR-07 | 1순위 | 작성자는 자신의 게시글을 수정할 수 있어야 한다. |
| FR-08 | 1순위 | 작성자는 자신의 게시글을 삭제할 수 있어야 한다. |
| FR-09 | 2순위 | 로그인 사용자는 게시글에 댓글을 작성할 수 있어야 한다. |
| FR-10 | 2순위 | 작성자는 자신의 댓글을 삭제할 수 있어야 한다. |
| FR-11 | 2순위 | 사용자는 제목/본문 키워드로 게시글을 검색할 수 있어야 한다. |
| FR-12 | 2순위 | 게시글 목록은 페이지네이션 또는 무한스크롤 방식으로 제공되어야 한다. |
| FR-13 | 3순위 | 사용자는 다크/라이트 테마를 전환할 수 있어야 한다. |

## 5. 비기능 요구사항

| ID | 우선순위 | 요구사항 |
| --- | --- | --- |
| NFR-01 | 필수 | 로그인 정보는 안전하게 저장되어야 한다. |
| NFR-02 | 권장 | 게시글 목록 조회는 평균 1초 이내 응답을 목표로 한다. |
| NFR-03 | 필수 | 테마 설정은 브라우저 저장소에 보관되어 새로고침 후에도 유지되어야 한다. |
| NFR-04 | 필수 | 기본적인 접근성 기준을 충족해야 한다(명확한 포커스, 대비). |

## 6. 제약 및 가정

- 브라우저 기반 웹 앱으로 제공한다.
